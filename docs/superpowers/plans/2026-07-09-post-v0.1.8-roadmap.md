# Post-v0.1.8 Roadmap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete v0.1.9 release merge, validate Play Store status, maintain codebase, and position for next feature cycle based on Play Store feedback.

**Architecture:** Three parallel work streams — (1) merge v0.1.9 PR and bump version, (2) monitor Play Store review and document response options, (3) clean up 40+ stale feature branches from MVP phase. All streams complete independently; #2 informs subsequent feature planning.

**Tech Stack:** Git CLI, GitHub CLI (gh), Google Play Console API (manual), Release notes documentation.

## Global Constraints

- Unpeeky is React Native + Expo on Android (Google Play only for now)
- All PRs require CI gates: typecheck, lint, format, test (GitHub Actions in .github/workflows/ci.yml)
- Version format: `major.minor.patch` (e.g., 0.1.9), updated in both app.json and android/gradle.properties
- Android versionCode must increment by 1 for each Google Play upload (currently at 8 in v0.1.8)
- Polish is primary language for user-facing strings (src/i18n/strings.ts)
- No breaking changes to domain logic or storage schema without migration tests
- Release notes are bilingual (Polish + English) in docs/RELEASE_NOTES_v0.X.Y.md

### Task 3: Clean Up Stale Feature Branches

**Files:**

- No source files changed (git operations only)
- Branches to delete: 40+ stale MVP-phase branches (feat/_, fix/_, chore/_, feature/_, docs/\*)

**Interfaces:**

- Consumes: Local git repository with all branches fetched
- Produces: Cleaned remote (stale branches deleted), local repo synced, branch list documented

- [ ] **Step 1: List all stale branches (older than 2026-06-01)**

```bash
git branch -a --sort=-committerdate | grep -E "feat/|fix/|chore/|feature/" | head -20
```

Expected: Shows branches from MVP phase

- [ ] **Step 2: Prune local tracking branches**

```bash
git fetch origin --prune
```

Expected: Removes local branches that no longer exist on origin

- [ ] **Step 3: Identify merged branches safe to delete**

```bash
git branch -r --merged main | grep -E "origin/(feat/|fix/|chore/|feature/)" | wc -l
```

Expected: Shows count of branches merged to main

- [ ] **Step 4: Batch delete merged branches from remote**

```bash
for branch in $(git branch -r --merged main | grep -E "origin/(feat/|fix/|chore/|feature/)" | sed 's|origin/||'); do
  git push origin --delete "$branch" 2>&1 | grep -E "deleted|error"
done
```

Expected: Multiple "[deleted]" lines

- [ ] **Step 5: Verify cleanup success**

```bash
git fetch origin --prune
git branch -a | grep -E "feat/|fix/|chore/|feature/" | wc -l
```

Expected: Significant reduction to near 0

- [ ] **Step 6: Document cleanup**

Create `docs/BRANCH_CLEANUP_LOG.md` describing what was deleted, when, and why.

- [ ] **Step 7: Commit cleanup documentation**

```bash
git add docs/BRANCH_CLEANUP_LOG.md
git commit -m "chore: document branch cleanup (40+ stale MVP branches deleted)

Removed feature/fix/chore branches from MVP phase that were merged to main.

Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>"
```
